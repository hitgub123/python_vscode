### 4. 出力

| 項目       | 内容                     | 型       | 備考                               |
| ---------- | ------------------------ | -------- | ---------------------------------- |
| アプリ連携状態 | アプリの連携状態（有効/無効） | 列挙型   |                                    |

### 5. 処理詳細

1.  システム管理者が、アプリ連携管理画面を開き、対象の従業員のアプリ利用許可設定を行う。
2.  システムは、入力されたアプリ利用許可設定（有効/無効）を取得する。
3.  取得した情報を基に、従業員マスタのアプリ連携状態を更新する。
4.  アプリ連携状態の更新が成功した場合、システム管理者に更新完了メッセージを表示する。

### 6. 例外処理

| 条件                                                                    | 対応                                                                                                                       |
| ----------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------- |
| 入力チェックエラー（アプリ利用許可設定の値が不正）           | <span style="color:red;">エラーメッセージをシステム管理者に表示し、設定処理を中断する。エラーログを出力する。</span>                                         |
| データベースへの書き込みエラー                                                        | <span style="color:red;">エラーメッセージをシステム管理者に表示し、設定処理を中断する。システム管理者に通知する。エラーログを出力する。</span>                         |

### 7. 画面設計

*   アプリ連携管理画面：
    *   従業員一覧
    *   アプリ利用許可設定（有効/無効）
    *   更新ボタン
    *   更新完了メッセージ表示領域
    *   エラーメッセージ表示領域

### 8. その他

*   セキュリティ：アプリ連携状態は、適切なアクセス制御を行い、不正な変更を防止する。
*   ログ：アプリ連携状態の変更履歴をログに記録する。
